In-class Exercises

Part 1: CSV Layer Projections

  1. Download today's data.
  2. Open Council District Boundaries in QGIS.
  3. Open NYPD Motor Vehicle Collisions (CSV) in QGIS.
  4. Open 7 Major Felony Incidents (CSV) in QGIS.
  5. All of the layers should line up. If they do not, remove the layers that do not line up with the council districts and reload them while paying extra attention to the projection.

Part 2: Metadata

  1. Look at the Street Centerline dataset on NYC Open Data.
  2. We're looking for an attribute in the data that describes the kind of street each feature is. Look at a few features' attributes by clicking on them and see if you can find one.
  3. Let's look at the metadata. Click the About tab, scroll down to Attachments, and download Centerline.pdf.
  4. Open the PDF.
  5. Which projection is the file in? Look under Spatial Reference.
  6. Which attribute would help explain which type each street is? Look under Fields.

Part 3: Categorized maps

  1. Open the streets shapefile in QGIS. You could continue with your style from the Class 2 in-class exercises if you like.
  2. Style using a Categorized style on the column RW_TYPE. Customize each category's style and label text using the metadata for the file.
  3. Open the NYPD Motor Vehicle Collisions CSV in QGIS if it's not already open.
  4. Style using a Categorized style on the column CONTRIBUTING FACTOR VEHICLE 1.
  5. Try categorizing on some other fields.

Part 4: Selecting data

  1. Open the NYPD Motor Vehicle Collisions CSV in QGIS if it's not already open.
  2. Use Select by expression to select only features where the NUMBER OF PERSONS INJURED field is over 0.
  3. Clear the selection.
  4. Use Select by expression to select only features where the BOROUGH field is equal to BRONX.
  5. Export the selected data as its own shapefile and open it.
  6. Experiment with using Select by expression with another field and/or dataset.
